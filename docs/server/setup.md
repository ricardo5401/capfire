# Server setup

Capfire server is a Rails 7.1 API-only app. It runs on ONE node (one systemd
unit per deploy host), talks to a Postgres database it owns, and exposes an
HTTPS API authenticated by JWT bearer tokens.

## Prerequisites

You are responsible for providing:

- **A Linux host** with systemd (Ubuntu 22.04+, Debian 12+ tested).
- **Ruby 3.2+** installed via RVM, rbenv, asdf, or the distro package.
  The installer detects whichever one you have.
- **PostgreSQL** reachable via a connection string. The installer calls
  `rails db:prepare` on an empty database.
- **A public TLS terminator** in front of Capfire — typically Nginx on the
  same box, or Cloudflare / an internal load balancer. SSE streams require
  a timeout of at least 1 hour.
- **sudo/root access** on the host.

Capfire does *not* install Ruby or Postgres for you. Those rabbit holes are
distro-specific; we let you pick the flavor you already use on your fleet.

## Quick install

From a fresh checkout of the repo:

```bash
git clone git@github.com:capfire-project/capfire.git
cd capfire
sudo ./scripts/install-server.sh \
  --database-url='postgres://capfire:pass@127.0.0.1:5432/capfire_production'
```

The installer will:

1. Create a `capfire` system user.
2. Rsync the source to `/opt/capfire`.
3. Write `bin-wrappers/ruby` and `bin-wrappers/bundle` pointing at whatever
   Ruby/Bundler your shell currently resolves to (so systemd doesn't need
   RVM/rbenv sourced anywhere).
4. Run `bundle install --deployment --without development test`.
5. Generate `/etc/capfire/env` with a random `CAPFIRE_JWT_SECRET`
   (`openssl rand -hex 64`) and the DATABASE_URL you passed.
6. Create the apps root at `/srv/apps`, owned by `capfire`.
7. Run `rails db:prepare`.
8. Install `/etc/systemd/system/capfire.service`, enable and start it.
9. Symlink `/usr/local/bin/capfire` → `/opt/capfire/bin/capfire` so the
   admin CLI is on PATH.

Installer flags:

| Flag | Default | Purpose |
|---|---|---|
| `--dir DIR` | `/opt/capfire` | Install prefix |
| `--user USER` | `capfire` | System user that runs the service |
| `--apps-root DIR` | `/srv/apps` | Where `capfire project add` clones apps |
| `--database-url URL` | *(prompted)* | DATABASE_URL for Rails |
| `--jwt-secret SECRET` | *(autogenerated)* | Override the random secret |
| `--skip-service` | off | Don't install/start the systemd unit |
| `--skip-bundle` | off | Don't run `bundle install` |
| `--skip-db` | off | Don't run `rails db:prepare` |
| `--reinstall` | off | Tolerate an existing `/opt/capfire` |

Once the service is up:

```bash
curl http://localhost:3000/healthz   # -> "ok"
sudo systemctl status capfire
sudo journalctl -u capfire -f
```

## Post-install checklist

**1. Create an admin token for yourself.**

```bash
sudo -u capfire capfire tokens create \
  --name=admin \
  --apps='*' \
  --envs='production,staging' \
  --cmds='deploy,restart,rollback,status,drain,restore'
```

Copy the printed JWT — the server will never show it again.

**2. Put a TLS terminator in front.**

See [nginx](#nginx-sse-proxy) below for the SSE-safe config.

**3. Add your first app.**

```bash
sudo -u capfire capfire project add git@github.com:myorg/myapp.git
```

Then configure a `capfire.yml` inside the clone. See
[per-app config](config.md).

## Nginx (SSE proxy)

SSE fails silently the moment any proxy in the chain buffers or applies a
60s idle timeout. Use:

```nginx
location /capfire/ {
    proxy_pass         http://127.0.0.1:3000/;
    proxy_http_version 1.1;
    proxy_set_header   Host $host;
    proxy_set_header   X-Real-IP $remote_addr;

    # MANDATORY for SSE — disable response buffering.
    proxy_buffering    off;
    proxy_cache        off;
    chunked_transfer_encoding on;

    # Allow long silences without dropping the connection. Real deploys
    # have no-stdout stretches of minutes (vite build, DB migrations,
    # asset rsync). Nginx default is 60s.
    proxy_read_timeout 3600s;
    proxy_send_timeout 3600s;
}
```

Capfire emits `X-Accel-Buffering: no` on every SSE response, but some setups
ignore it — set `proxy_buffering off` explicitly.

> **Cloudflare in front.** CF has its own idle timeout (100s Free/Pro,
> 600s Business, up to 6000s Enterprise). Options:
>
> 1. Use a DNS record with the proxy off ("grey cloud") for the Capfire hostname.
> 2. Rely on Capfire's built-in SSE heartbeats every 15s. Verify with DevTools
>    that `: keep-alive` lines arrive.

## Manual install (without the script)

If your environment requires custom steps, run the installer with
`--skip-*` flags or follow it piecewise:

```bash
sudo useradd --system --create-home --shell /bin/bash capfire
sudo rsync -a --exclude='.git' --exclude='client/' /path/to/repo/ /opt/capfire/
sudo chown -R capfire:capfire /opt/capfire

sudo -u capfire bash -lc 'cd /opt/capfire && bundle config set --local deployment true && bundle config set --local without "development test" && bundle install'

sudo mkdir -p /etc/capfire
sudo tee /etc/capfire/env >/dev/null <<EOF
RAILS_ENV=production
CAPFIRE_JWT_SECRET=$(openssl rand -hex 64)
DATABASE_URL=postgres://capfire:pass@localhost:5432/capfire_production
CAPFIRE_APPS_ROOT=/srv/apps
EOF
sudo chown root:capfire /etc/capfire/env
sudo chmod 0640 /etc/capfire/env

sudo mkdir -p /srv/apps
sudo chown capfire:capfire /srv/apps

sudo -u capfire bash -lc 'cd /opt/capfire && set -a && . /etc/capfire/env && set +a && RAILS_ENV=production bundle exec rails db:prepare'

# Render the systemd unit (see scripts/templates/capfire.service)
sudo cp scripts/templates/capfire.service /etc/systemd/system/capfire.service
sudo sed -i -e 's|__CAPFIRE_USER__|capfire|g' \
            -e 's|__CAPFIRE_HOME__|/opt/capfire|g' \
            -e 's|__CAPFIRE_APPS_ROOT__|/srv/apps|g' \
            /etc/systemd/system/capfire.service
sudo systemctl daemon-reload
sudo systemctl enable --now capfire
```

## Uninstall

```bash
sudo ./scripts/uninstall-server.sh           # removes unit + symlink
sudo ./scripts/uninstall-server.sh --purge   # also wipes /opt/capfire + /etc/capfire
```

The uninstaller never touches your database or the apps root — both can
hold user data.
